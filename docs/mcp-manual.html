Manual MCP GTI

Guia pratico para conectar o MCP de dominio do GTI em Claude Desktop e Codex, usar as ferramentas de tarefas, KPIs e OKRs, e operar com seguranca usando token de sessao.

GTI Claude Desktop Codex CLI / IDE MCP HTTP remoto

Ele serve para desktop ou precisa ser agente de codigo?

O MCP do GTI agora tem dois modos: HTTP remoto para usuarios finais e stdio local para desenvolvimento. O modo recomendado para usuarios comuns e o remoto, porque nao exige repositorio, Node, build nem arquivo `.env`.

Cliente Funciona? Observacao
Claude Desktop Sim Use a URL remota quando o cliente suportar MCP HTTP. O modo local continua disponivel para desenvolvimento.
Codex CLI Sim Use `config.toml` com `url` e header Authorization para o endpoint remoto.
Codex IDE extension Sim CLI e extensao IDE compartilham configuracao MCP; o endpoint remoto evita setup local.
Codex desktop app Depende do suporte MCP disponivel na sua versao Depende do suporte a MCP HTTP/remote da versao instalada. Se o app aceitar URL MCP e bearer token, use o endpoint remoto.

Em outras palavras: ele nao precisa ser usado apenas dentro de um agente de codigo. O ideal de produto e usa-lo como conector MCP remoto normal, autenticado como o usuario logado no GTI.

Conexao remota recomendada

Para usuarios finais, use este endpoint: 

https://go-gti.vercel.app/api/mcp

O cliente MCP deve enviar `Authorization: Bearer ACCESS_TOKEN_COPIADO_DA_UI`. Esse token e curto e representa o usuario logado, entao o Supabase continua aplicando RLS.

Codex remoto

[mcp_servers.gti]
url = "https://go-gti.vercel.app/api/mcp"
http_headers = { Authorization = "Bearer ACCESS_TOKEN_COPIADO_DA_UI" }

Claude Code remoto

claude mcp add --transport http gti "https://go-gti.vercel.app/api/mcp" --header "Authorization: Bearer ACCESS_TOKEN_COPIADO_DA_UI"

Este e o MVP remoto com Bearer token curto. A evolucao ideal e OAuth/PKCE, para que o cliente abra login GTI e armazene credenciais sem o usuario copiar token.

Pre-requisitos locais

Esta secao e apenas para desenvolvimento local do MCP.

  1. Repositorio GTI clonado em `C:\Users\Notebook\Desktop\gti`.
  2. Node instalado.
  3. Dependencias instaladas com `npm install`.
  4. Servidor MCP compilado com `npm run mcp:build`.
  5. Token de sessao atual copiado pela UI do GTI.
cd C:\Users\Notebook\Desktop\gti
npm install
npm run mcp:build
npm run mcp:smoke

Token de sessao

O MCP usa `GTI_MCP_USER_ACCESS_TOKEN`, que e o access token da sessao do usuario logado no GTI. No Supabase, esse access token normalmente expira em cerca de 1 hora.

Nao copie `refresh_token` para o MCP. O refresh token renova sessoes e tem risco maior. O GTI copia somente o access token curto.

Como copiar

  1. Abra `https://go-gti.vercel.app/settings`.
  2. No card MCP, clique em `Atualizar sessao`.
  3. Copie `URL remota`, `Codex`, `Claude Code` ou `Token`.
  4. Se o card avisar token expirado/perto de expirar, atualize antes de copiar.

.env local recomendado para desenvolvimento

VITE_SUPABASE_URL=https://SEU_PROJETO.supabase.co
VITE_SUPABASE_ANON_KEY=SUA_ANON_KEY
GTI_MCP_USER_ACCESS_TOKEN=ACCESS_TOKEN_COPIADO_DA_UI

Claude Desktop

Claude Desktop usa um arquivo JSON de configuracao MCP. O caminho exato varia por sistema/instalacao, mas o formato do servidor GTI e este: 

{
  "mcpServers": {
    "gti": {
      "command": "node",
      "args": [
        "C:\\Users\\Notebook\\Desktop\\gti\\mcp\\dist\\server.js"
      ],
      "env": {
        "VITE_SUPABASE_URL": "https://SEU_PROJETO.supabase.co",
        "VITE_SUPABASE_ANON_KEY": "SUA_ANON_KEY",
        "GTI_MCP_USER_ACCESS_TOKEN": "ACCESS_TOKEN_COPIADO_DA_UI"
      }
    }
  }
}

Fluxo

  1. Compile o MCP: `npm run mcp:build`.
  2. Copie token/snippet novo no GTI.
  3. Atualize o JSON do Claude Desktop.
  4. Reinicie o Claude Desktop.
  5. Peca: "Use o MCP GTI e chame `gti_healthcheck`".

Codex

O Codex documenta suporte a MCP stdio na CLI e na extensao IDE. A configuracao fica em `config.toml`, normalmente em `~/.codex/config.toml`, ou em `.codex/config.toml` dentro de um projeto confiavel.

Opcao 1: config.toml

[mcp_servers.gti]
command = "node"
args = ["mcp/dist/server.js"]
cwd = "C:\\Users\\Notebook\\Desktop\\gti"

[mcp_servers.gti.env]
VITE_SUPABASE_URL = "https://SEU_PROJETO.supabase.co"
VITE_SUPABASE_ANON_KEY = "SUA_ANON_KEY"
GTI_MCP_USER_ACCESS_TOKEN = "ACCESS_TOKEN_COPIADO_DA_UI"

Opcao 2: carregar do ambiente local

Como o servidor MCP do GTI tambem le `.env` quando inicia na raiz do projeto, voce pode deixar o token no `.env` local e configurar so comando/cwd: 

[mcp_servers.gti]
command = "node"
args = ["mcp/dist/server.js"]
cwd = "C:\\Users\\Notebook\\Desktop\\gti"

Features do MCP GTI

Contexto

  • `gti_get_site_context`
  • Recursos `gti://context/site` e `gti://context/tasks`

Referencias

  • Status, categorias e projetos
  • Perfis e busca de pessoas por nome/email

Tarefas

  • Listar, buscar e abrir detalhes
  • Criar, atualizar, mover status e arquivar com confirmacao
  • Checklist e comentarios

Resumos

  • Minhas tarefas
  • Projeto e categoria
  • Sugestoes de proximas acoes

KPIs

  • Listar KPIs
  • Abrir KPI
  • Resumo semanal fora da meta
  • Planos de acao

OKRs

  • Listar objetivos
  • Key results
  • Marcos e progresso calculado

Prompts prontos

Contexto inicial

Use o MCP do GTI. Leia o contexto do site e me diga quais ferramentas estao disponiveis antes de agir.

Tarefas urgentes

Liste minhas tarefas urgentes abertas no GTI. Traga titulo, status, projeto, prazo e progresso de checklist.

Resumo operacional

Resuma minhas tarefas no GTI. Traga contagens por status/prioridade, tarefas atrasadas, proximas de vencer e progresso medio de checklist.

Proximas acoes

Sugira proximas acoes para minhas tarefas no GTI. Use apenas tarefas reais, cite os IDs e explique o criterio usado. Nao crie tarefas, nao mova status e nao comente automaticamente.

Criar tarefa

Crie uma tarefa no GTI com titulo "Revisar SLA transportadora", prioridade medium e status <statusId>. Antes, confirme que os IDs de status/projeto existem.

Atribuir para outra pessoa

Procure o perfil de Maria pelo MCP e, se encontrar um unico resultado, crie uma tarefa atribuida a ela. Use o ID retornado pela busca e inclua confirmAssignedToOther=true.

KPIs

Liste os KPIs fora da meta na semana atual e destaque quais estao sem valor informado.

OKRs

Liste os OKRs do GTI com progresso por objetivo, key result e marcos.

Guardrails

  • O MCP age como o usuario do token `GTI_MCP_USER_ACCESS_TOKEN`.
  • Nao usa `service_role`.
  • Leituras e escritas respeitam RLS do Supabase.
  • Arquivar tarefa exige `confirmArchive=true`.
  • Deletar checklist exige `confirmDelete=true`.
  • Mover para status final exige `confirmFinalize=true`.
  • Trocar responsavel exige `confirmOwnerChange=true`.
  • Criar tarefa para outra pessoa exige `confirmAssignedToOther=true`.
  • Ferramentas de sugestao nao escrevem nada automaticamente.
  • Ferramentas de KPI/OKR atuais sao somente leitura.

Validacao

npm run mcp:build
npm run mcp:smoke
npm run mcp:check
npm run mcp:permissions

`mcp:permissions` precisa de tokens separados de `member`, `lead` e `admin` para validar permissoes reais. Sem esses tokens, ele pula os papeis e explica o que falta.

Troubleshooting

Erro Causa comum Como resolver
`GTI_MCP_USER_ACCESS_TOKEN is required` Token ausente no `.env` ou config MCP. Copie novo snippet no GTI e reinicie o cliente MCP.
`token is expired` Access token passou de aproximadamente 1 hora. Atualize sessao no card MCP, copie novo token e rode `npm run mcp:smoke`.
Servidor nao aparece no cliente Caminho errado, build ausente ou cliente nao reiniciado. Rode `npm run mcp:build`, use caminho absoluto no Claude e reinicie.
Ferramenta pede confirmacao Acao sensivel. Repita com a flag correta apenas apos confirmacao humana.
